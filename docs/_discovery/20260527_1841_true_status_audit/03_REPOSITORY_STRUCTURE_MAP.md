## Codebase Analysis

### 1. Generated Artifacts
- **SDK Typings:** `packages/sdk/src/generated/api.d.ts` is generated from `packages/contracts/openapi.yaml`.
- **Prisma Client:** Autogenerated into `node_modules/.prisma/client` and used across the backend apps (`api` and `worker`).

### 2. Obsolete or Temporary Folders
- **Old Discovery Audits:** 
  - `docs/_discovery/20260325T230126Z/`
  - `docs/_discovery/20260505_0858_full_system_runtime_audit/`
  - `docs/_discovery/20260505_0900_full_system_runtime_audit/`
  These are snapshots of previous audits and should be archived or excluded from production deployments.
- **Mock Gateway Fixtures:**
  - `docs/mocks/fixtures/`
  These files are used by the mock Caddy resolve script and Dev UI Mock gateway. They are not active in production.

  - `apps/api/src/opd/` (controllers and services for Out-Patient Department)
  Scaffolding exists in the backend API to prepare for OPD module integration, but the UI is not yet built. This code is currently inactive but must remain in place to guarantee LIMS/OPD extensibility.
